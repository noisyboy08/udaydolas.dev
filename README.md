## Overview

This repository powers [udaydolas.dev](https://udaydolas.dev) - a minimal yet expressive portfolio for Uday Dolas. It blends a case-study driven blog, an interactive component registry, LLM-ready docs, and a polished personal profile backed by Next.js 15, React 19, and Tailwind CSS v4.

### Why this project exists

- Showcase engineering work, experiments, and speaking engagements
- Share reusable UI/UX building blocks via a public shadcn registry
- Publish long-form MDX posts with gorgeous code blocks and OG images
- Provide recruiter-friendly resources (vCard, JSON-LD, RSS, `/llms.txt`)

---

## Highlights

- **Multi-section layout**: Profile, projects, timeline, blog, and components live under one cohesive design system.
- **Content authoring**: MDX-powered blog (`/src/content/blog`) with automatic RSS feed, OG image generation, and keyboard shortcuts.
- **Component registry**: Self-hosted registry (`/src/registry`, `/src/__registry__`) consumable via the shadcn CLI for quick sharing of effects, hooks, and UI primitives.
- **LLM + SEO**: Custom `/llms.txt`, JSON-LD, structured metadata, sitemap, robots, and RSS under `src/app`.
- **PWA ready**: Manifest, icons, metadata, and theme color helpers for installable experiences.
- **Accessibility-focused**: Semantic components, Radix primitives, keyboard navigation, and reduced-motion friendly animations.
- **Developer ergonomics**: Turbopack builds, pnpm scripts, strict ESLint/TypeScript configs, registry automation, and capture scripts.

## Tech Stack

- **Framework**: Next.js 15 (App Router, Turbopack)
- **Language**: TypeScript 5.7, React 19, modern Server/Client Components
- **Styling**: Tailwind CSS v4, custom fonts, utility helpers
- **UI System**: shadcn/ui + Radix primitives with bespoke enhancements
- **Content**: MDX, `remark/rehype`, dynamic OG image routes
- **Tooling**: pnpm, ESLint 9, Prettier, Husky, Zod, SWR, Jotai, shiki
- **Deploy Target**: Vercel (first-class), but portable to any Node host

## Getting Started

### Prerequisites

- Node.js 20 (or >= 22)
- pnpm 9+
- Git

### Local setup

```bash
git clone https://github.com/noisyboy08/udaydolas-portfolio.git
cd udaydolas-portfolio

pnpm install
cp .env.example .env.local
# fill in optional secrets (analytics, site overrides, etc.)

pnpm dev
```

Visit `http://localhost:1408` for the dev server.

### Production build

```bash
pnpm build
NODE_ENV=production pnpm start
```

---

## Environment Variables

Use `.env.local` to override values. Reference `.env.example` for available keys (analytics, base URLs, API keys). Never commit secrets; `.env*` files are already gitignored.

## Content Authoring

- **Blog posts**: add MDX files under `src/content/blog`. Assets live in `public/images/blog`. Routes auto-register and feed into `/blog`, `/rss`, and OG image generation under `src/app/og`.
- **Components / Registry**: add entries under `src/registry`. Run `pnpm registry:build` to refresh `src/__registry__/registry.autogenerated.json` so consumers can `npx shadcn@latest add https://udaydolas.dev/r/<component>.json`.
- **LLM briefs**: update markdown in `src/app/(llms)`; `/llms.txt` is generated for AI agents.
- **Profile data**: adjust `src/data/user.ts` and structured data under `src/features/profile/data`.

## Deployment Notes

- Optimized for Vercel (image optimization, streaming, edge-friendly routes).
- OG image builders live in `src/app/og` for rich sharing previews.
- PWA metadata comes from `src/config/site.ts`, `src/app/manifest.ts`, and theming helpers in `src/lib`.
- SEO surfaces: `src/app/robots.ts`, `src/app/sitemap.ts`, `src/app/rss/route.ts`, and `src/app/vcard/route.ts`.
